# ¿Cómo funcionan las experiencias?

---

## Flujo general

Se detecta el login + carrera del usuario.

Si hay un login existente, se muestra la sección del formulario
y la experiencia propia en el modal de experiencias. Si no hay login, ambas secciones se ocultan (display: none).

Para traer los datos de afuera, al pulsar uno de los botones de experiencias en majors se guarda
una carrera (El titulo de majors) en una variable dependiendo el botón que se pulse (`software`, `gestion`, `inicial` y `primaria`).
Con esa variable ya se sabe qué experiencias traer.

---

## Función — Compartir tu experiencia

En el formulario se guardan los datos; el backend hace el resto.

Cosas a considerar:
- Por estudiante debe haber una única experiencia. Una vez que se publica, no dejar que el estudiante publique otra.
- En el caso de usar mi BD, el nombre y apellido se tienen que unificar (se mandan separados, el backend los une).

---

## Función — Editar / Borrar

Al tener una experiencia publicada, se utilizan las funciones
ya preparadas que guardan los datos necesarios en la sección del Backend.

---

## Ubicacion de las funciones

Las funciones estarian preparadas al final del .js

## Funcionalidades extras

- Límite de experiencias que se traen de golpe, para que no colapse nada.
  Por ejemplo, una función que traiga las experiencias a medida que el scroll baja por debajo del 80%.

---

## Conclusión

Fuera de la sección del Backend, solo hay que tratar de conseguir el usuario y, si es posible,
la carrera a la que pertenece el mismo, para no dejarlo publicar en una carrera ajena.
Lo demás ya está preparado en la sección del Backend. Guardar los datos en las variables especificadas

---

## Mapa de cambios — ¿Qué tocar cuando el backend trae datos de otro lado?

Cuando el backend esté listo, el único cambio es reemplazar el contenido de `experiences_modal.js` actual
por el contenido del archivo `experiences_modal_backend.js` que ya está preparado (copiar y pegar).

Ese archivo ya tiene los datos necesarios y no depende del JSON local para nada.

---

### data-loader.js

Sacar la llamada a `loadExperiencesData()` dentro de `loadAllData()` ya que el JSON de experiencias
no se va a usar más:

```js
// Antes
static async loadAllData() {
    await Promise.all([
        DataLoader.loadAppData(),
        DataLoader.loadExperiencesData()   // ← sacar esta línea
    ]);
}
```

También se pueden sacar `loadExperiencesData() (la funcion que esta leyendo el json)`, 
`getExperiencesData()` y `waitForExperiencesData()` del archivo si se quiere dejar todo limpio, pero no es obligatorio.
