Como funcionan las experiencias?

Se detecta el login

Si hay un login existente, por ejemplo, en el aula virtual, te muestra la seccion del formulario,
si no, la oculta (display: none).

Para traer los datos de afuera, al pulsar en uno de los botones de experiencias en majors se guarda
una carrera en una variable dependiendo el boton que se pulse (software, gestion, inicial y primaria).
Con la misma ya se sabe que experienicas traer.

Funcion Compartir tu experiencia:
En el formulario se guardan los datos y se envian, el backend hace lo demas.
Cosas a considerar:
- Por estudiante debe haber una única experiencia, una vez que se publica no dejar que el estudiante publique otra.
- La misma se tendra que mostrar en algun lugar para poder editarla.

Funcion Editar/Borrar:
Al tener una experiencia publicada (te la mostraria si detecta un login), se necesitara un modal emergente tomando
aparte el ID que trae el propio json y lo guarda en "window.ExperiencesData = data". Talvez la experiencia propia
haya que traerla aparte.

Funcionalidades extras:
- Limite de experiencias que se traen de golpe, para que no colapse nada (por ejemplo, una funcion que traiga
las experiencias a medida que el scroll baja por debajo del 80%).
